🎓 2. Installation de PostgreSQL

Bienvenue dans la section installation ! C’est ici que PostgreSQL va sortir du monde des idées pour s’inviter dans votre ordinateur. Pas de panique : vous n’avez pas besoin d’être sorcier ni DevOps certifié pour réussir cette étape.

Dans ce tutoriel, vous allez apprendre à installer PostgreSQL sur Windows, macOS et Linux — en suivant la voie classique (l’installateur officiel)… ou la voie des ninjas (via Docker 🥷)

🖥️ Installer PostgreSQL sur Windows

🧭 Introduction

PostgreSQL est un SGBDR (Système de Gestion de Base de Données Relationnelle) puissant, robuste et libre. Il est né sur des terres UNIX, mais il a appris à parler Windows couramment.

Dans cette leçon, vous apprendrez à :

  • installer PostgreSQL sur un PC Windows,
  • configurer votre environnement comme un pro,
  • et préparer le terrain pour les leçons à venir.

En somme, vous allez poser les fondations solides de tout le reste.

🌐 Alternatives à l’installation locale

Envie d’éviter toute installation ?
Vous êtes plutôt du genre cloud & chill ? 😎

Bonne nouvelle : des solutions hébergées existent. Par exemple, la plateforme Neon vous permet d’avoir un PostgreSQL 100 % cloud, gratuitement, sans installer quoi que ce soit, et sans donner votre numéro de carte. Un clic, et vous êtes en ligne.

Idéal pour tester rapidement… ou impressionner vos collègues à la pause café.

🔽 Étape 1 – Télécharger l’installateur PostgreSQL pour Windows

Rendez-vous sur la page de téléchargement officielle de PostgreSQL, hébergée par EnterpriseDB :
téléchargement de PostgreSQL.

Là, téléchargez la dernière version stable pour Windows 64 bits.

💡 Au moment où ce tutoriel a été rédigé, la version la plus récente était PostgreSQL 17.5-1. Si une version plus récente est proposée, foncez : le futur, c’est maintenant.

🧙 Étape 2 – Lancer l’installation étape par étape

⚠️ Droits d’administrateur requis

PostgreSQL, c’est pas une petite appli. C’est un système serveur complet. Pour cette raison, son installation fait des choses sérieuses :

  1. 📁 Crée des dossiers dans C:\Program Files\PostgreSQL\...
  2. 👤 Ajoute un compte système spécial (postgres)
  3. 🔄 Installe un service Windows qui démarre PostgreSQL automatiquement
  4. 🛠️ Modifie les variables d’environnement (comme PATH)
  5. 🔐 Initialise les permissions et la sécurité de la base

🧪 Alternatives pour utilisateurs avancés

Envie de faire les choses autrement ? Voici deux approches un peu plus techniques — mais aussi plus flexibles.

✅ 1. Installation portable (sans installation système)

Vous pouvez télécharger une version “portable” de PostgreSQL, sans l’installer au sens classique du terme. Il faudra ensuite :

  1. exécuter initdb pour initialiser le cluster,
  2. lancer le serveur avec pg_ctl,
  3. ne pas casser Windows (promis).

⚠️ À réserver aux utilisateurs expérimentés. (Si ce paragraphe vous fait peur, vous n’en avez probablement pas besoin.)

✅ 2. Utiliser PostgreSQL via Docker ou WSL
  1. 🐳 Docker : PostgreSQL tourne dans un conteneur isolé. Magique, rapide, propre.
  2. 🐧 WSL (Windows Subsystem for Linux) : vous faites tourner PostgreSQL… dans Linux… depuis Windows. Oui, c’est aussi étrange que puissant.

👉 Ces solutions nécessitent Docker ou WSL préinstallés… ce qui, ironiquement, demande souvent les droits d’administrateur au départ. 🫠

🟢 En résumé
L’installation classique de PostgreSQL sous Windows nécessite des droits d’administrateur.

Mais si vous êtes du genre rebelle technique, vous avez aussi des options portables ou virtualisées (Docker, WSL). À chacun son style.

🔸 Étape 1 – Lancer la fenêtre de contrôle de compte utilisateur

Vous avez téléchargé fièrement le fichier postgresql-17.5-1-windows-x64.exe depuis le site officiel ? Parfait. Double-cliquez dessus et… BOOM 💥 — Windows sort sa carte “sécurité maximale”.

🛡️ Contrôle de compte utilisateur (UAC)

Une fenêtre bleue (ou grise) s’affiche avec un message du genre :

“Voulez-vous autoriser cette application d’un éditeur inconnu à apporter des modifications à votre appareil ?”

Pas de panique. Ce n’est pas une attaque alien, c’est juste Windows qui fait son travail.

⚙️ L’installateur de PostgreSQL a besoin de modifier des éléments système :
  • Créer des dossiers
  • Ajouter un service
  • Configurer votre environnement

Et pour ça, il demande poliment… votre bénédiction administrative.

✅ Que faire ici ?

✔️ Si vous avez bien téléchargé le fichier depuis le site officiel de PostgreSQL / EnterpriseDB, cliquez sur “Oui” sans hésiter

🟥 Si le bouton “Oui” est grisé, c’est que vous n’avez pas les droits administrateur. ➡️ Dans ce cas, faites appel à un administrateur… ou au collègue qui a toutes les clés du royaume Windows.

🟢 En résumé
✅ Cette étape ne lance pas encore l’installation, mais donne à PostgreSQL les super-pouvoirs nécessaires pour fonctionner correctement.

Une fois que vous cliquez sur “Oui”, l’installateur se lance dans une nouvelle fenêtre, prêt à vous accompagner pas à pas comme un bon coach de sport… mais pour les bases de données 😄.

🔸 Étape 2 – Accueil de l’assistant d’installation de PostgreSQL

Après avoir vaillamment cliqué sur “Oui” dans la fenêtre de contrôle de compte utilisateur, vous êtes officiellement accueilli par… l’assistant d’installation de PostgreSQL 🎉

Une fenêtre classique s’affiche, avec ce bon vieux bouton “Suivant >” (ou “Next >” si vous êtes en mode Windows anglais).

🧙 Que se passe-t-il à cette étape ?

C’est le point de départ de toute l’installation.
L’assistant vous annonce sobrement (mais fermement) qu’il va vous accompagner dans un voyage en 11 étapes — sans dragons, mais avec quelques composants bien utiles.

Ce n’est pas encore le moment de faire des choix techniques, juste de se dire : “OK, allons-y.”

🤓 Bon à savoir

À ce stade :

  • Rien n’a encore été copié ni installé
  • Vous pouvez encore fermer la fenêtre si vous avez peur de l’engagement (mais on vous jugera un tout petit peu 😅)
  • Le bouton “Suivant” vous fera entrer dans le vrai tunnel d’installation
🟢 En résumé
🟨 L’assistant PostgreSQL vient de s’ouvrir.
Cliquez simplement sur “Suivant” pour continuer.

Vous entrez maintenant dans le vif du sujet. Plus moyen de reculer (enfin si, mais chut).

🔸 Étape 3 – Choisir le dossier d’installation

Vous avez cliqué sur “Suivant” ?
Bravo, vous venez de franchir la porte numéro 2 du grand jeu de l’installation PostgreSQL 🎉
Vous voilà maintenant face à une nouvelle question existentielle (version informatique) :

🗂️ Dans quel dossier souhaitez-vous installer PostgreSQL ?

📍 Chemin proposé par défaut

L’assistant vous propose généralement : C:\Program Files\PostgreSQL\17\

C’est ici que seront stockés :

  • Le serveur PostgreSQL lui-même (le cœur du système),
  • Les outils en ligne de commande (psql, etc.),
  • L’interface graphique pgAdmin,
  • Et tous les autres petits fichiers qui font battre le cœur de la bête.

❓Puis-je changer ce dossier ?

Oui. Techniquement, vous pouvez l’installer dans :

  • D:\PostgreSQL\
  • C:\BaseDeDonnéesDeLaMort\
  • Ou même Z:\ChâteauSecret\PostgreSQL\

Mais… sauf si vous avez une raison technique (espace disque, SSD dédié, partition chiffrée, bunker souterrain 😄…), il est fortement recommandé de garder le chemin par défaut.

🔍 Pourquoi ? Parce que :

  • Ce dossier est standardisé,
  • Les futures mises à jour s’y attendent,
  • Et vous évitez bien des soucis pour plus tard.
⚠️ Attention
Ce dossier n’est pas celui où PostgreSQL stockera vos bases de données.
C’est juste là où les fichiers du programme seront installés.
Le dossier de données (bases, logs, etc.) sera défini à l’étape suivante.
🟢 En résumé
🟨 Cette étape vous permet de définir le répertoire d’installation du serveur PostgreSQL.
🧠 Sauf besoin particulier, laissez le chemin par défaut. Ça simplifie la vie… et ça évite les migraines.

➡️ Cliquez ensuite sur “Suivant” pour passer à l’étape 4 : la sélection des composants à installer — une étape où l’on coche des cases, littéralement (et symboliquement aussi).

🔸 Étape 4 – Sélectionner les composants à installer

Après avoir choisi où PostgreSQL va s’installer, vous tombez sur l’écran de sélection des composants.

C’est un peu comme faire ses courses dans un supermarché… mais au rayon “base de données” 🛒

Et la bonne nouvelle ?
👉 Tous les bons ingrédients sont déjà sélectionnés pour vous par défaut. 😄

🧩 Composants proposés

Vous verrez une liste de cases à cocher, comme ceci :

Composant Description
PostgreSQL Server Le cœur du système — le moteur de base de données. 🎯 Indispensable.
pgAdmin 4 L’interface graphique web intuitive pour gérer vos bases. Très pratique. 🧑‍💻
Command Line Tools Les outils en ligne de commande comme psql, createdb, dropdb, etc.
StackBuilder Un outil pour télécharger des modules complémentaires (PostGIS, ODBC, etc.). Facultatif.
Documentation Une copie locale de la documentation officielle. Utile hors ligne.

✅ Que faire ici ?

En général : ne touchez à rien ! Toutes les cases sont cochées par défaut, et c’est exactement ce qu’il faut pour démarrer sur de bonnes bases.

Mais si vous êtes du genre aventurier :

  • Vous pouvez décocher StackBuilder (peu utilisé si vous ne comptez pas installer de modules externes)
  • Ou la documentation locale (si vous avez une connexion Internet constante)
🔧 Spoiler : la majorité des utilisateurs gardent tout tel quel — et ça fonctionne parfaitement.
🟢 En résumé
🟨 Cette étape vous permet de choisir les composants à installer.
✅ Pour la plupart des utilisateurs, gardez les options par défaut — elles couvrent tous les besoins courants.

➡️ Une fois satisfait de votre sélection (ou si vous n’avez rien changé), cliquez sur “Suivant”. Vous passerez alors à l’étape 5 : le choix du dossier de données, là où PostgreSQL stockera vos bases.

🔸 Étape 5 – Choisir le dossier pour les données

Après avoir sélectionné les composants à installer, l’assistant PostgreSQL vous invite à définir le dossier des données.

C’est dans ce répertoire que le système stockera :

  • Les bases de données 📚
  • Les fichiers de configuration (genre postgresql.conf, pg_hba.conf) ⚙️
  • Les journaux de transactions (les logs, pas les tickets de caisse 🧾)
  • Et tout ce que PostgreSQL considère comme vital à son bon fonctionnement.

📁 Répertoire par défaut

Par défaut, le chemin proposé ressemble à ceci :
C:\Program Files\PostgreSQL\17\data\

Ce dossier sera créé automatiquement si vous continuez l’installation. C’est un peu le cœur battant de votre serveur PostgreSQL. ❤️‍🔥

🧠 Peut-on modifier ce chemin ?

Oui. Mais attention, vous entrez dans une zone à responsabilité élevée :

Vous pouvez changer si... Mais attention à...
💾 Vous avez un disque dur séparé dédié aux données 📌 Ne jamais déplacer ce dossier après installation
🧑‍💻 Vous travaillez en entreprise avec des exigences IT ❌ Ne jamais choisir un dossier temporaire ou réseau
💼 Vous avez besoin d’une structure spécifique (D:\DB) 🚨 Ne jamais supprimer ce dossier manuellement ensuite
🔴⚠️ À ne jamais faire
  • Ne stockez pas les données sur un disque amovible (USB, disque réseau…)
  • Ne mettez pas ce dossier dans Documents, Téléchargements ou un dossier synchronisé (genre OneDrive)
  • Et surtout : ne le renommez pas après l’installation (PostgreSQL risquerait la crise cardiaque)
🟢 En résumé
🟨 Cette étape vous permet de choisir où seront stockées toutes vos données PostgreSQL.
✅ Si vous ne savez pas quoi choisir, gardez le chemin proposé par défaut. Il est sûr, standardisé et pratique.
⚠️ Attention
Ce dossier est ultra sensible : ne le touchez pas une fois PostgreSQL installé, sauf si vous savez très exactement ce que vous faites.

➡️ Cliquez ensuite sur “Suivant” pour passer à l’étape suivante, et non des moindres : la définition du mot de passe du superutilisateur postgres. 😎

🔸 Étape 6 – Définir le mot de passe du superutilisateur postgres

Excellent 👌 — on passe maintenant à une étape décisive de l’installation de PostgreSQL : celle où vous devenez officiellement le super-héros (ou superutilisateur) de votre base de données 🦸‍♀️🦸‍♂️

L’assistant d’installation vous demande maintenant de renseigner un mot de passe pour l’utilisateur postgres.

Et non, ce n’est pas une simple formalité :
👉 ce mot de passe vous permettra d’ouvrir les portes du royaume PostgreSQL 👑
(et si vous l’oubliez… eh bien, bon courage ! 😅)

👤 Qui est postgres ?

postgres est le superutilisateur de votre serveur PostgreSQL.
Il a tous les droits, tous les pouvoirs, et… toutes les responsabilités 🧠

Avec ce compte, vous pouvez :

  • Créer et supprimer des bases de données
  • Ajouter ou retirer des utilisateurs
  • Modifier les permissions, les schémas, les extensions, etc.
  • (Et même faire planter le serveur si vous jouez trop avec le feu 🔥)

🔐 Ce que vous devez faire

L’assistant vous propose deux champs à remplir :

  • Mot de passe : saisissez un mot de passe robuste
  • Confirmer le mot de passe : tapez-le une seconde fois pour éviter les fautes de frappe
⚠️ Attention
PostgreSQL ne vous proposera aucune option de récupération.
Si vous oubliez ce mot de passe, vous devrez passer par des procédures manuelles complexes.

✅ Conseils pour un bon mot de passe

À faire ✔️ À éviter ❌
Utilisez 8 à 16 caractères Mot de passe vide
Combinez lettres, chiffres, symboles “123456”, “admin”, “password”
Exemple : Jobe@2025 Mots simples ou personnels

💡 Petit rappel : Ce mot de passe sera utilisé par pgAdmin, psql et tous vos scripts Conservez-le en lieu sûr, ou utilisez un gestionnaire de mots de passe.

🟢 En résumé
🟨 Cette étape vous demande de définir le mot de passe du superutilisateur postgres — le compte le plus puissant du serveur.
🔐 Choisissez un mot de passe sérieux, fort et inoubliable (ou bien noté quelque part).
❌ Ne le perdez pas, ne l’oubliez pas, ne le sous-estimez pas. Il est la clé de tout.

➡️ Cliquez sur “Suivant” lorsque votre mot de passe est bien défini. Nous passerons à l’étape 7 :

le choix du port d’écoute de PostgreSQL, autrement dit : "Par où votre serveur entendra les appels".

🔸 Étape 7 – Choisir le port d’écoute de PostgreSQL

On avance sereinement vers une étape un peu plus technique, mais ô combien importante : le choix du port.
Ou pour le dire autrement :

“À quelle porte doit-on frapper pour parler à PostgreSQL ?” 🚪🎙️

Une fois votre mot de passe défini, l’assistant vous invite à configurer le port d'écoute de PostgreSQL. Pas de panique, ce n’est pas une ligne téléphonique ni un port maritime. 👉 Il s'agit simplement d’un canal de communication numérique par lequel les applications vont dialoguer avec votre serveur PostgreSQL

📡 Le port d’écoute, c’est quoi au juste ?

Un port est un numéro compris entre 1 et 65535, que les logiciels utilisent pour s’écouter les uns les autres.

PostgreSQL, par tradition et par défaut, utilise le port 5432.

Donc quand vous vous connectez à votre base de données, vous le faites généralement à : localhost:5432

Traduction :

  • localhost = votre propre ordinateur
  • 5432 = le port où PostgreSQL est à l’écoute 👂

🎯 Dois-je changer ce numéro ?

Dans 90 % des cas, laissez 5432.
Mais voici quelques scénarios où vous pourriez vouloir le modifier :

Cas de modification Exemple de nouveau port
Vous avez déjà une autre instance PostgreSQL 5433, 5434
Vous voulez faire du multi-serveur 54321, 15432, etc.
Vous avez un conflit avec un autre logiciel N’importe quel port libre
Vous êtes parano et aimez brouiller les pistes 🕵️‍♂️ 65000 par exemple
⚠️ Attention :
Si vous changez le port ici, vous devrez toujours l’indiquer lors de vos connexions, comme ceci :
psql -U postgres -h localhost -p 5433.
Sinon, psql ira chercher au 5432… et se prendra un mur 🧱
🟢 En résumé
🟨 Cette étape vous permet de définir le port d’écoute de PostgreSQL.
✅ Gardez 5432 sauf si :
  • Vous avez déjà un autre serveur PostgreSQL
  • Vous avez un besoin spécifique
  • Vous aimez les numéros exotiques
🎯 Si vous modifiez le port, notez-le bien. Il faudra l’utiliser pour chaque connexion.

➡️ Cliquez sur “Suivant” pour accéder à une étape un peu méconnue mais essentielle :

le choix des paramètres régionaux (locale) — ou comment votre base trie, date et ordonne les choses.

🔸 Étape 8 – Choisir les paramètres régionaux (Locale)

Cette étape passe souvent inaperçue, mais elle a un impact subtil et durable sur le fonctionnement de votre base de données. Bienvenue dans le monde mystérieux des… paramètres régionaux 🧙‍♂️🌍

Une fois le port d’écoute défini, l’assistant PostgreSQL vous invite à sélectionner une locale.

Et non, ce n’est pas une option pour choisir votre marché local du dimanche matin 🧺.
Ici, locale signifie : paramètres culturels de votre base de données.

🧭 C’est quoi une “locale” dans PostgreSQL ?

La locale détermine le comportement linguistique et régional de PostgreSQL, notamment :

  • Le tri alphabétique (ex. : est-ce que "école" vient avant ou après "étude" ?)
  • La casse des lettres (Majuscules/minuscules dans les comparaisons)
  • Les formats de date et d’heure
  • Les séparateurs décimaux (virgule ou point ?)
  • Et plus généralement, la sensibilité linguistique de la base

🧙 Que propose l’assistant PostgreSQL ici ?

Vous verrez un menu déroulant intitulé Locale.
Par défaut, il est réglé sur : Default locale
Ce qui signifie :

“Utilise les paramètres régionaux de ton système Windows”
(ex : fr_FR.UTF-8 si votre Windows est en français, ou en_US.UTF-8 pour un système anglophone)

🤔 Dois-je modifier cette locale ?

Dans 95 % des cas : Non. Gardez la locale par défaut.
Mais vous pouvez en choisir une autre si vous :

Situation Exemple de locale à choisir
Travaillez avec des données multilingues en_US.UTF-8, C, POSIX
Utilisez des outils anglophones qui s’attendent à l’anglais en_US.UTF-8
Avez besoin d’un tri binaire sans sensibilité linguistique C ou POSIX
Déployez une base partagée entre plusieurs fuseaux/équipes en_GB.UTF-8, fr_FR.UTF-8, etc.

🧪 Si vous êtes débutant, pas d’inquiétude. La valeur par défaut fonctionne très bien dans presque tous les cas.

🟢 En résumé
🟨 La locale détermine comment PostgreSQL trie, compare, date et chiffre les choses.
✅ Laissez Default locale si vous ne savez pas quoi choisir.
Cela convient parfaitement pour un usage personnel, professionnel ou en formation.
🧙‍♂️ Les utilisateurs avancés peuvent choisir une locale spécifique pour des besoins linguistiques précis.

➡️ Cliquez maintenant sur “Suivant” pour accéder à l’avant-dernière étape :

le résumé de vos choix d’installation, sorte de dernier point de non-retour avant le grand plongeon 💾💥

🔸 Étape 9 – Vérifier le résumé de l’installation

Nous voilà à l’étape du grand récapitulatif — celle où PostgreSQL vous regarde droit dans les yeux et vous demande :
“On y va, ou tu veux encore changer un truc ?” 😏

Toutes les cases sont cochées ✅
Tous les champs sont remplis 🧾
Votre destin est (presque) scellé. Mais avant de lancer la machine, PostgreSQL vous présente un dernier écran intitulé :

Pre Installation Summary
(ou Résumé avant installation)

C’est votre dernière chance de relire vos choix avant de tout lancer. Comme un plan de vol juste avant le décollage. ✈️

📋 Que contient cette page ?

C’est le moment de revoir tous vos choix, et peut-être de repérer une petite coquille avant qu’il ne soit trop tard.

La fenêtre affiche une liste détaillée de 11 éléments, chacun correspondant à une étape que vous avez configurée.

💡 Si quelque chose vous semble incorrect (par exemple, vous avez choisi un port déjà utilisé par votre machine à café connectée ☕), cliquez sur Retour pour revenir à l’étape concernée et corriger.

Parce que c’est votre dernière chance de vérifier avant que les fichiers soient copiés, les services windows démarés… et le destin de votre base soit scélé à jamais. Ce qui revient à invoquer toute la puissance de PostgreSQL (même les pingouins de Lunix 🐧 en tremblent.)

🔍 Que devez-vous faire ici ?

  • 👁️ Relisez attentivement chaque ligne de ce résumé
  • 🔁 Revenez en arrière si une option vous semble douteuse
  • ✅ Validez si tout vous paraît correct
⚠️ Astuce de prudence :
Il vaut mieux cliquer sur “Retour” maintenant que regretter un mauvais paramètre plus tard (comme un mauvais dossier, un port erroné, ou un mot de passe oublié).
🟢 En résumé
🟨 Cette étape vous montre en un coup d’œil toutes les décisions que vous avez prises jusqu’ici. C’est une vue d’ensemble de tous les paramètres choisis pendant l’installation.
C’est le moment de les relire soigneusement, avant de lancer l’installation.
🔄 Besoin de changer quelque chose ? Aucun souci : utilisez les boutons “Retour” pour corriger ce qu’il faut.
👁️ Si tout est bon, respirez profondément, et cliquez sur “Suivant” avec la satisfaction du travail bien fait.

➡️ Vous passerez alors à l’étape suivante — parfois affichée — intitulée “Prêt pour l’installation”, avant d’entrer dans le vif du sujet : l’installation proprement dite de PostgreSQL 🎉

🔸 Étape 10 – Prêt pour l’installation

Vous pensiez que l’installation allait commencer tout de suite après le résumé ?
Eh bien… pas encore ! 😄

PostgreSQL vous réserve une dernière étape de confirmation, une sorte de soupir final avant le grand saut. Cette fenêtre s’intitule Prêt pour l’installation

🧠 À quoi sert cette étape ?

Ce n’est pas une formalité vide. Cette page a plusieurs objectifs bien utiles :

  • ✅ Vous informer que toutes les options ont été validées
  • 🔁 Vous laisser une ultime chance pour revenir en arrière
  • 🛡️ Vous éviter de lancer l’installation par mégarde
  • 🚀 Vous donner le signal que l’installation réelle va enfin commencer

C’est en quelque sorte le moment où PostgreSQL vous regarde droit dans les yeux et vous demande :

"Tu es bien sûr de vouloir faire ça ?"
Car après… il n’y aura plus de retour. 🫣
💬 Le message exact affiché est généralement :
"l’assistant d’installation est maintenant prêt à démérrer l’istallation de PostgreSQL sur votre ordinateur."

🔍 Que devez-vous faire ici ?

Pas de configuration à changer.
Seulement lire, réfléchir une dernière fois… et cliquer.

⚠️ Conseil rapide : Si vous réalisez que quelque chose ne vous semble pas correct (chemin, port, composant oublié, mot de passe incertain), c’est le bon moment pour cliquer sur le bouton “Back”.

Ce bouton est là pour vous sauver avant qu’il ne soit trop tard. Et il fonctionne.
Oui, même à cette étape !

🟢 En résumé
🟨 Cette étape est la porte d’entrée vers l’installation réelle.
Tout est prêt, validé, empaqueté… il ne reste qu’à lancer.
🎯 Si tout vous semble en ordre, cliquez sur “Suivant” pour commencer l’installation proprement dite.
Et respirez : PostgreSQL s’occupe de tout. (Enfin… presque.)

🔸 Étape 11 – Installation de PostgreSQL en cours

C’est le moment que vous attendiez.

Vous avez validé tous vos choix, cliqué sur “Suivant”, et maintenant, PostgreSQL se met au travail.
L’écran qui s’affiche porte généralement ce titre :
Installation en cours”

🛠️ Que se passe-t-il exactement ?

Le programme d’installation exécute automatiquement une série d’opérations techniques pour mettre PostgreSQL en place sur votre machine.

Voici ce qu’il fait en coulisses (pendant que vous fixez la barre de progression 👀) :

🧩 Étapes techniques réalisées :

  1. 📁 Création des dossiers d’installation
    (ex. : C:\Program Files\PostgreSQL\17)
  2. 📥 Copie des fichiers nécessaires
    (serveur, utilitaires, pgAdmin, etc.)
  3. 🛠️ Configuration automatique des fichiers internes
    (postgresql.conf, pg_hba.conf, etc.)
  4. 🗃️ Initialisation du cluster de base de donnéesPostgreSQL
  5. 👤 Création du compte système postgres (si nécessaire)
  6. 🔄 Ajout et démarrage du service Windows PostgreSQL
  7. Écriture dans le registre Windows (pour le menu Démarrer, les services, etc.)

🕐 Durée de l’installation

⏳ En général, cette étape prend entre 1 et 3 minutes sur une machine moderne.
Pendant ce temps, vous pouvez :

  • faire une pause café ☕,
  • lancer une méditation rapide 🧘‍♂️,
  • ou simplement contempler la barre verte en silence.

🔴 Que faire en cas d’erreur ?

Bien que rares, certaines erreurs peuvent survenir :

Problème fréquent Cause probable
❌ Port déjà utilisé Vous avez une autre instance PostgreSQL active
❌ Échec de création de service Droits administrateur insuffisants
❌ Fichier bloqué Antivirus ou pare-feu trop protecteur
💡 Si cela vous arrive :
  • Notez le message d’erreur affiché,
  • Fermez l’assistant,
  • Corrigez la cause (ex. : redémarrez, changez de port, désactivez temporairement l'antivirus),
  • Puis relancez l’installation.

🛠️ Cette étape correspond à la vraie phase d’installation.
PostgreSQL se déploie, se configure, se lance.
Et vous, vous n’avez rien à faire… sauf patienter.

🎉 Une fois terminé, vous serez automatiquement redirigé vers la dernière étape :
la confirmation de la réussite… et peut-être le lancement de Stack Builder.
